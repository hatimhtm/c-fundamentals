<p align="center">
  <em>A reference implementation of the algorithms, data structures, and systems primitives every C programmer eventually writes by hand. Five sorts, binary search, two ciphers, a linked list, a hash table, a cross-platform sysinfo tool — all in C11, zero dependencies, GCC + Clang clean on Linux + macOS, and 227 unit-test assertions to keep them honest.</em>
</p>

---

### `/// WHAT'S INSIDE`

| Module | Highlights | Binary |
|---|---|---|
| **Sorting** | Selection · Insertion · Bubble · Quicksort (Lomuto) · Merge sort. Both `int` and `char**` variants for every algorithm. | `build/sorting` |
| **Searching** | Binary search — iterative and recursive — with size_t-safe midpoint. | `build/bsearch` |
| **Encryption** | Caesar with chi-squared frequency-analysis cracker · Vigenère cipher (encrypt + decrypt). | `build/crypto-cli` |
| **Data structures** | Singly-linked list (push front/back, find, reverse) · Open-addressing string→int hash table with djb2, linear probing, tombstones, auto-resize. | `build/linked-list-demo`, `build/hash-table-demo` |
| **Systems** | Cross-platform `sysinfo` using POSIX `uname`, `getpwuid`, sysctl on macOS, sysconf on Linux. | `build/sysinfo` |
| **Benchmarks** | Empirical comparison of all five sorts on `n=100`, `1000`, `10000`. Markdown-formatted output. | `build/benchmark` |

### `/// QUICK START`

```bash
git clone https://github.com/hatimhtm/c-fundamentals.git
cd c-fundamentals
make all      # build every binary into build/
make test     # 227 assertions across 43 cases
make smoke    # run every CLI once with sample input
```

Wants Clang? `make clean && make all CC=clang`. Custom flags? `make all CFLAGS='-Wall -Wextra -Werror -std=c11 -O3'`.

### `/// SORTING`

```bash
./build/sorting --algo=quick zebra ant mouse cat
# Before: ["zebra", "ant", "mouse", "cat"]
# After : ["ant", "cat", "mouse", "zebra"]
# Algorithm: quick

# Pipe stdin
echo -e "foo\nbar\nbaz" | ./build/sorting --algo=merge
```

Algorithms: `selection · insertion · bubble · quick · merge`. Each is in its own `.c` file with the same `*_ints` / `*_strings` interface.

### `/// SEARCHING`

```bash
./build/bsearch 7 3 1 9 7 4 2
# Sorted: [1, 2, 3, 4, 7, 9]
# Target: 7
# Result: found at index 4
```

Sorts the input via `insertion_sort_ints` (small array, simple), then runs binary search. Both iterative and recursive variants live in [`binary_search.c`](algorithms/searching/binary_search.c) — they're tested for equivalence on every value 1..21 against a known sorted array.

### `/// ENCRYPTION`

```bash
# Caesar
./build/crypto-cli -e 3 "Hello World"            # Khoor Zruog
./build/crypto-cli -d 3 "Khoor Zruog"            # Hello World
./build/crypto-cli -c "Khoor Zruog"              # → Guessed shift: 3 (chi-squared)

# Vigenère
./build/crypto-cli -v --key=lemon "attack at dawn"   # lxfopv ef rnhr
./build/crypto-cli -V --key=lemon "lxfopv ef rnhr"   # attack at dawn
```

The Caesar cracker uses chi-squared distance against standard English letter frequencies (not the naive "most-common letter == E" approach). Test [`caesar_crack_recovers_shift`](tests/test_caesar.c) exercises it on real text.

### `/// DATA STRUCTURES`

```bash
./build/linked-list-demo 1 2 3 4 5
# Built (size=5): [1 → 2 → 3 → 4 → 5]
# find(1): index 0
# Reversed (size=5): [5 → 4 → 3 → 2 → 1]

./build/hash-table-demo the cat sat on the mat
# Counts (size=4, capacity=16):
#   the                  2
#   cat                  1
#   sat                  1
#   on                   1
#   mat                  1
```

The hash table uses **open addressing with linear probing** + djb2 hashing + tombstone deletion + auto-resize at 0.75 load factor. Capacity is always a power of two so the modulo collapses to a bitmask. See [`hash_table.c`](data_structures/hash_table/hash_table.c).

### `/// SYSINFO`

```bash
./build/sysinfo -v
# System Information
# ----------------------------------------
#   OS:             Darwin
#   Architecture:   arm64
#   ...
```

### `/// TESTING`

The test runner is hand-rolled — no external framework. `tests/test.h` defines `ASSERT`, `ASSERT_EQ_INT`, `ASSERT_EQ_STR`, `ASSERT_EQ_SIZE`, `ASSERT_TRUE/FALSE`, `ASSERT_NULL/NOT_NULL`. Each test increments global counters; failures print location + message but **do not abort** so a single `make test` run reports every failure across every suite.

```
$ make test
…
────────────────────
Total assertions: 227
Failures:         0
✓ All tests passed.
```

Coverage:
| Suite | Cases |
|---|---|
| `test_sorts` | All 5 algorithms, both variants, edge cases (empty / 1 / sorted / reversed) |
| `test_search` | Iterative · recursive · empty · missing · strings |
| `test_caesar` | Encrypt · decrypt · negative shift · large shift · punctuation · round-trip · chi-squared crack |
| `test_vigenere` | Textbook example · case preserved · round-trip · empty key · punctuation passthrough |
| `test_linked_list` | Create · push front/back · find · reverse · pop empty |
| `test_hash_table` | Create · set/get · overwrite · remove · resize-rehash · slot reuse after remove |

### `/// CONTRIBUTING`

Pull requests welcome. The code style is K&R-ish, two-space indent, snake_case, doc comments on every public function. Please:

1. Keep `-Wall -Wextra -Werror -pedantic` clean on both GCC and Clang.
2. Add tests for new code — at minimum, one positive case + one edge case.
3. Don't introduce dependencies. If you need a hash function, write it inline.
